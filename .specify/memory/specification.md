# 项目规格说明书 (Project Specification)

## 项目概述 (Project Overview)

### 项目名称
能源云需求进度管理产品 (Energy Cloud Requirements Progress Management Product)

### 项目愿景
打造一个全面、准确、实时的能源云需求进度管理平台，实现数据驱动的项目管理和决策支持。

### 核心需求描述
从公司飞书项目和数仓中全面且准确地同步各个需求的关键数据，并以此为基础，统计关联项目的预算和进度以及关联产品的推进程度，最后以仪表盘的形式展现出来。

## 本体设计

### 对象的定义和关系

一共有三个主要的对象，分别是需求，项目，产品。
一个产品就是一个智能的软件。
一个需求是某个软件需要添加的某个新功能。
一个项目是几个具有相同目的的需求集合。因为一个目的可能需要多个产品配合，所以一个项目会涉及多个产品。
总结，一个产品对应多个需求。一个项目也对应多个需球，这些需求一定属于某个产品，但可能属于多个产品。

## 功能需求 (Functional Requirements)

### 1. 数据同步模块

#### 1.1 飞书项目同步需求数据
- **数据源**: 飞书项目API
- **同步内容**: 所有需求的所有核心数据
- **同步频率**: 每日凌晨定时全量同步
- **数据验证**: 完整性检查、重复数据去重

#### 1.2 飞书表格同步项目和产品数据
- **数据源**: 飞书文档API
- **同步内容**: 所有项目和产品的核心数据（不含"每周实际填报工时"）
- **同步频率**: 每周二凌晨定时全量同步
- **数据验证**: 完整性检查、重复数据去重

#### 1.3 数仓同步项目工时数据
- **数据源**: 公司数据仓库
- **同步内容**: 项目每周实际填报工时（人天）
- **同步频率**: 每周二凌晨定时全量同步
- **数据验证**: 一致性检查、异常值检测

### 2. 原始数据可视化模块

#### 2.1 需求明细数据展示
- 列表展示所有需求数据
- 支持搜索、筛选、分页、导出
- 支持字段自定义（隐藏、排序、位置调整）
- 不支持详情查看、编辑、新增
- **本地备注**: 每个条目支持人工编辑的本地备注（不受同步影响）

#### 2.2 项目明细数据展示
- 列表展示所有项目数据
- 支持搜索、筛选、分页、导出
- 支持字段自定义（隐藏、排序、位置调整）
- 支持详情查看，展示：
  - 分产品预算明细
  - 工时明细（每周增量 + 累计总工时）
  - 项目指标（见下文）
- 不支持编辑、新增
- **本地备注**: 每个条目支持人工编辑的本地备注

#### 2.3 产品明细数据展示
- 列表展示所有产品数据
- 支持搜索、筛选、分页、导出
- 支持字段自定义（隐藏、排序、位置调整）
- 支持详情查看，展示：
  - 总预算（可能来自不同项目汇总）
  - 产品指标（见下文）
- 不支持编辑、新增
- **本地备注**: 每个条目支持人工编辑的本地备注

### 3. 数据分析模块

#### 3.1 项目指标计算

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| **项目完成率** | 项目下所有需求预期开发总人天 / 项目总预算人天 | 反映需求规模与预算匹配度 |
| **预算使用率** | 项目累计总工时 / 项目总预算人天 | 反映预算消耗进度 |
| **时间进度** | 已用时间 / 项目计划时间（天） | 反映时间进展 |
| **风险等级** | 见下表 | 综合评估项目健康度 |

**风险等级定义**:
- **无风险**: 项目完成率 ≥ 时间进度 **且** 预算使用率 ≤ 时间进度
- **高风险**: 项目完成率 ≤ 时间进度的70% **或** 预算使用率 ≥ 时间进度的130% **或** 预算使用率 ≥ 项目总预算的95%
- **低风险**: 其他情况

#### 3.2 产品指标计算

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| **产品上市周期** | 产品下所有需求上市周期的平均值 | 反映产品迭代速度 |
| **需求池水位** | 评审通过但未宣讲的需求个数 | 反映待处理需求积压 |

> **需求上市周期**: 需求从启动到实际上线所经历的天数

### 4. 通知模块

#### 4.1 风险告警
- **告警规则**:
  - 低风险项目：界面高亮提示
  - 高风险项目：界面高亮 + 飞书机器人告警
- **通知方式**: 飞书机器人
- **通知频率**: 实时

#### 4.2 定期汇报
- **汇报内容**: 项目整体健康度、产品整体需求池水位
- **通知方式**: 飞书机器人
- **通知频率**: 每周

## 信息架构 (Information Architecture)

### 交互风格

#### 设计理念
- **风格定位**: B端管理后台风格，强调数据密度和信息效率
- **设计原则**:
  - **清晰层级**: 通过间距、色彩、字号建立清晰的信息层级
  - **效率优先**: 减少操作步骤，支持批量操作和快捷操作
  - **数据可视化**: 复杂数据通过图表、色块、标签直观呈现
  - **一致性**: 相同功能保持相同的交互模式和视觉表现

### 组件交互规范

#### 1. 数据表格 (Table)

**基础规范**
- 表头: 14px, 600字重, `#262626`, 背景`#FAFAFA`
- 行高: 48px
- 斑马纹: 偶数行背景`#FAFAFA`
- 边框: 仅底部边框`1px solid #F0F0F0`
- 对齐: 文本左对齐，数字右对齐，操作列居中

**交互行为**
| 场景 | 交互 | 说明 |
|------|------|------|
| Hover行 | 背景变为`#F5F5F5` | 整行响应 |
| 选中行 | 左侧4px主色条 + 背景`#E6F4FF` | 单选/多选支持 |
| 排序列 | 表头显示上下箭头图标 | 点击切换升/降序 |
| 列宽调整 | 拖拽列边缘调整 | 实时预览宽度 |
| 横向滚动 | 固定首列 + 表头 | 大数据表场景 |

**字段自定义**
- 点击表头右侧"设置"图标打开配置面板
- 面板以抽屉形式从右侧滑出 (宽度320px)
- 支持: 显示/隐藏字段、拖拽排序、固定列设置
- 配置自动保存到用户偏好

**分页器**
- 位置: 表格右下角
- 默认页码: 1, 2, 3...10, 跳转到
- 每页条数: 可选 10/20/50/100
- 总条数显示: "共 128 条"

#### 2. 搜索与筛选

**全局搜索**
- 位置: 标题栏右侧，用户头像左侧
- 样式: 圆角输入框，`border-radius: 20px`
- 占位符: "搜索需求、项目、产品..."
- 交互: 输入实时搜索，延迟300ms防抖
- 下拉: 显示最近5条搜索历史

**筛选器**
- 位置: 表格上方，与操作按钮同行
- 样式: 胶囊形状筛选标签 + "更多筛选"按钮
- 已选筛选项: 以标签形式展示，可点击×移除
- 快速筛选: 状态、时间范围、负责人等常用条件外露
- 高级筛选: 点击"更多"展开筛选面板

**筛选面板**
- 展开方式: 向下展开，覆盖下方内容
- 布局: 2-3列网格，每列一个筛选项
- 操作: 底部固定"重置"和"确定"按钮
- 预设方案: 支持保存常用筛选组合

#### 3. Tab 切换

**样式**
- 类型: 线型Tab (Underline Tabs)
- 高度: 40px
- 激活态: 底部2px主色线 + 文字主色
- 未激活: 文字`#595959`

**交互**
- 点击切换，无页面刷新
- 支持手势滑动 (移动端)
- Tab过多时显示左右箭头滚动

**主页Tab结构**
```
[需求] [项目] [产品]
      ↑
   默认选中"需求"
   切换时保留当前筛选条件
```

#### 4. 按钮规范

| 类型 | 样式 | 用途 | 尺寸 |
|------|------|------|------|
| 主按钮 | 主色填充背景 + 白色文字 | 主要操作、提交 | 高32px，左右内边距16px |
| 次按钮 | 白色背景 + 主色边框 | 次要操作、取消 | 同上 |
| 文字按钮 | 纯文字 + hover变色 | 表格操作、链接 | 无内边距 |
| 图标按钮 | 仅图标 | 工具栏操作 | 32x32px |
| 危险按钮 | 红色背景/边框 | 删除、风险操作 | 同主按钮 |

**按钮状态**
- Hover: 亮度变化或背景色加深
- 点击: scale(0.98) 微缩放反馈
- 加载: 显示Spinner，禁用点击
